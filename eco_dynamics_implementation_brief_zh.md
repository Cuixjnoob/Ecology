# 部分观测生态动力学项目：实现整理稿

## 1. 项目一句话

这个项目要解决的问题是：在**只能观测部分物种轨迹**的条件下，学习一个具有生态结构约束的动力学模型，使它不仅能做下一步预测，还能在**多步正向模拟 rollout** 中保持稳定并优于仅观测基线。

第一版模型不追求“识别真实隐藏物种”，而是要学出一个**潜在生态通道**，用来承接那些无法仅靠观测物种直接相互作用解释的动力学残差。

---

## 2. 文档真正要求我们做什么

原规范的核心不是“做一个更大的模型”，而是做一个**最小但自洽**的实现。必须保留的模块只有以下几类：

1. 延迟嵌入，用于短时历史重构；
2. 观测节点图骨架，保留物种级表示；
3. 潜在隐藏效应节点，用于承接未解释动力学；
4. 基于 GNN 的交互更新；
5. 从图状态到动力学增量的解码器；
6. 轻量级生态先验与稳定性正则；
7. 以 rollout 为中心的训练和评估。

不建议在第一版额外引入大型 Transformer、复杂神经 ODE、单独的仿真引擎或其他重型模块。第一版的关键词不是“最强表达能力”，而是“结构清楚、训练稳定、评估严谨”。

---

## 3. 必须守住的科学与实现边界

### 3.1 可以主张的内容

- 潜在生态通道能够提升多步 rollout 保真度；
- 这种提升在多个 horizon 和随机种子下持续存在；
- 这种提升不只是一般模型容量扩张带来的表面收益。

### 3.2 不能过度主张的内容

- 不能说模型已经识别出真实隐藏物种；
- 不能说边权就是真实生态因果网络；
- 不能把潜在节点和真实生物实体一一对应；
- 不能把良好的 rollout 解释为唯一可识别性已经成立。

这部分非常重要，因为它决定了后续实验报告和论文式表述的边界。

---

## 4. 最小可行版本（MVP）配置

建议直接把第一版固定成下面这个配置，避免在早期把搜索空间开得过大：

- 输入预处理：`log1p + z-score`
- 标准化统计量：只用训练集计算
- 延迟嵌入：`tau = 1`，`L = 6`
- 历史窗口：`W = 12` 或 `16`
- 潜在隐藏节点数：`N_h = 1`
- 节点嵌入维度：`d = 64`
- 观测编码器：小型 MLP
- 潜在节点初始化：由全局上下文 MLP 给出
- 图结构：稠密有向图 + 可学习边门控
- 消息传递层数：`2`
- 解码目标：预测增量 `delta x`
- 解码器：2 层 MLP
- 优化器：`AdamW`
- 学习率：`1e-3`
- weight decay：`1e-4`
- gradient clipping：`1.0`
- rollout curriculum：从 `H=1/2` 逐步增加到 `5/10/20`

这个版本已经足够覆盖原规范的全部主线目标。

## 5. 整个模型的逻辑主线

### 5.1 输入不是单时刻，而是一段历史

模型不应只看 `x_t`，而要看截至当前时刻的一个短窗口历史。原因有两个：

- 生态系统在部分观测条件下存在隐藏记忆；
- 延迟嵌入可以把这类隐藏效应留下的投影痕迹暴露出来。

### 5.2 观测物种必须保留为显式节点

这是原规范非常强调的一点。模型不能把所有观测物种压成一个黑箱向量再做预测，否则后面的边结构分析、潜在中介分析、物种级可视化都会丢掉。

### 5.3 潜在节点表示“聚合隐藏效应”

潜在节点的意义是功能性的，而不是字面生物学对象。它承担三类角色：

- 存储未解析生态记忆；
- 中介延迟或间接作用；
- 拟合 observed-only 模型无法解释的结构性残差。

### 5.4 预测的是增量，不是绝对状态

模型输出应该是 `delta x`，然后用

`x_{t+1} = x_t + delta x`

更新状态。这样更符合生态动力学中“速率 / 增量”的建模直觉，也更有利于训练稳定。

## 6. 模块拆解

### 6.1 数据与预处理

职责：

- 读取原始时间序列；
- 执行 `log1p` 变换；
- 用训练集统计量做 z-score；
- 保留反变换接口，供原始丰度空间评估使用。

注意点：

- 不能泄漏未来信息；
- train / val / test 必须按时间切分；
- 所有统计量必须只从 train 段估计。

### 6.2 延迟嵌入模块

对每个观测物种构造长度为 `L` 的延迟向量。若 `tau=1`，那么每个节点的输入就是：

`[x_t, x_{t-1}, x_{t-2}, ..., x_{t-L+1}]`

这一模块本质上是“局部时间重构器”，不是严格的 Takens 定理实现。

### 6.3 观测节点编码器

输入单个物种的延迟向量，输出该物种节点嵌入。建议使用小型 MLP 即可，不需要复杂时序编码器。

### 6.4 全局上下文编码器

把所有物种的延迟特征 flatten 后拼成全局上下文，再可选拼接环境协变量。这个全局上下文主要用于：

- 初始化潜在隐藏节点；
- 辅助边门控和边特征计算；
- 作为解码器的共享上下文。

### 6.5 潜在隐藏节点初始化

用全局上下文通过 MLP 生成一个潜在节点表示。第一版直接使用 `N_h = 1`，不要一开始就做多个隐藏节点分工。

### 6.6 图构建与边门控

边类型应覆盖：

- observed -> observed
- observed -> hidden
- hidden -> observed
- hidden -> hidden（当 `N_h > 1` 时）

第一版可用稠密图，再对每条边学习一个 `sigmoid gate alpha_ij`。这样实现最稳，也方便后续可视化平均交互矩阵。

### 6.7 GNN 消息传递

每条边生成消息，乘以门控后聚合到目标节点，再由更新函数得到新节点状态。推荐做法：

- 两层 message passing；
- 节点更新使用残差连接；
- 可选 LayerNorm；
- 不推荐 BatchNorm。

### 6.8 解码器

对每个观测节点，用“更新后的观测节点状态 + 潜在节点状态 + 紧凑上下文”预测下一步增量。第一版先做统一输出头，不要急着拆 direct / latent 两路输出。

### 6.9 生态正则项

第一版只保留轻量级正则，不做硬编码生态方程。建议包含：

- range penalty：抑制数值爆炸和越界；
- smoothness：抑制不合理的高频剧烈振荡；
- sparse penalty：弱化完全稠密图；
- metabolic prior：只有在有可信性状变量时才开启。

---

## 7. 前向传播应如何执行

单步前向的标准流程应该固定为：

1. 读取窗口末端为 `t` 的历史；
2. 预处理当前历史缓冲区；
3. 为每个物种构造延迟向量；
4. 编码观测节点；
5. 从全局上下文初始化潜在节点；
6. 构建边特征与边门控；
7. 执行 GNN 消息传递；
8. 解码得到 `delta x`；
9. 更新得到 `x_{t+1}`；
10. 若在 rollout 模式下，将预测写回缓冲区继续向前。

训练期 rollout 和推理期 rollout 应使用同一套逻辑，不要再维护一套独立仿真代码。

---

## 8. 损失函数设计

总损失应由以下几部分组成：

- 一步预测损失；
- rollout 损失；
- range 稳定性惩罚；
- 稀疏正则；
- 代谢先验正则；
- 平滑性正则。

推荐的起始权重可以直接采用原规范建议：

- `lambda_1 = 1.0`
- `lambda_2 = 0.5`
- `lambda_3 = 0.05`
- `lambda_4 = 1e-4`
- `lambda_5 = 0.0`
- `lambda_6 = 0.01`

这里最关键的一条是：**调参优先看 rollout 指标，不要被一步预测误差牵着走。**

---

## 9. 训练策略

### 9.1 数据切分

必须按时间顺序划分：

- train：前 60% 到 70%
- validation：中间 15% 到 20%
- test：最后 15% 到 20%

### 9.2 训练样本

每个样本由两部分组成：

- 历史窗口 `W`
- 未来 horizon `H`

形式上就是：

- 输入：`[t-W+1, ..., t]`
- 目标：`[t+1, ..., t+H]`

### 9.3 Curriculum

建议严格采用三阶段训练：

1. 阶段 A：只做一步 teacher forcing；
2. 阶段 B：做短 rollout，保留部分 teacher forcing；
3. 阶段 C：做目标 horizon 上的自由 rollout。

原规范反复强调，这一步对数值稳定性非常关键。

### 9.4 优化细节

- optimizer：`AdamW`
- lr：`1e-3`
- weight decay：`1e-4`
- batch size：`16` 到 `64`
- grad clip：`1.0`
- early stopping：看验证集 rollout 指标

---

## 10. 评估标准必须怎么做

### 10.1 主评估不是一步预测，而是 rollout

真正要回答的问题是：给模型一段真实历史后，让它自主向前模拟，它能否比 observed-only 模型更稳、更准地追踪未来动力学。

### 10.2 评估视野

至少要测三个 horizon：

- 短期：`5`
- 中期：`10`
- 长期：`20` 或数据允许的最大安全视野

### 10.3 至少要报告的指标

- rollout RMSE
- rollout MAE
- 逐物种 Pearson / Spearman
- 轨迹漂移得分
- 稳定性失败率

若条件允许，还可以加：

- 低维相空间距离
- 转折点准确率

### 10.4 必须对比的基线

至少要包含四组：

1. Persistence baseline
2. 仅观测 MLP / RNN
3. 仅观测图模型
4. 完整模型：观测图 + 潜在隐藏节点

如果完整模型在 rollout 上没有明显优于“仅观测图模型”，那么关于潜在生态通道的主张就站不住。

---

## 11. 消融实验的真正目的

原规范的消融不是装饰，而是为了验证每个设计是否真的有功能贡献。

### A1 去掉潜在隐藏节点

检验潜在生态通道是否必要。

### A2 去掉延迟嵌入

检验短时历史重构是否重要。

### A3 阻断潜在中介路径

保持容量接近，但切断 hidden -> observed 路径，检验作用是否真的来自“中介机制”而不是参数变多。

### A4 去掉 rollout loss

检验 rollout-aware 训练是否必要。

### A5 去掉生态正则

检验稳定性和生态合理性是否显著下降。

### A6 修改隐藏节点数量

测试 `N_h = 1, 2, 4`，判断一个聚合潜在节点是否已经足够。

## 12. 可视化交付清单

至少应产出以下图：

### V1 真值轨迹 vs rollout 轨迹

这是最重要的定性图，必须能直接看出模型是否发生漂移。

### V2 误差随 horizon 增长曲线

用来展示不同模型的误差随视野增长的速度。

### V3 逐物种 rollout 误差热图

帮助识别哪些物种真正从潜在通道中受益。

### V4 平均边门控热图

分别展示 observed-observed 和 hidden-observed 区块，但解释时必须克制，不能把它直接说成真实生态网络。

### V5 潜在活动强度 vs 残差代理

如果仅观测模型在某些区间明显失效，而潜在节点活动同步增强，这会是非常有说服力的结果。

### V6 相图比较

对于有振荡或循环动力学的数据，这个图很有价值，建议保留。

## 13. 常见失败模式与对应处理

### 13.1 一步预测很好，但 rollout 很差

说明模型只学会了局部拟合，没有学会稳定动力学。优先处理方式：

- 增加 rollout loss 权重；
- 拉长 rollout curriculum；
- 加强 range penalty；
- 适度减小模型容量。

### 13.2 潜在节点几乎没有被使用

表现通常是 hidden -> observed 门控接近零，或者完整模型与 observed-only 图模型几乎没有差别。优先检查：

- observed-only 基线残差是否真的存在结构性缺口；
- 直接路径是否过强；
- 潜在路径的解码器是否过弱。

### 13.3 潜在节点吸收一切

如果 hidden 路径垄断预测，而 observed-observed 边全部塌缩，说明模型失去结构解释性。可以尝试：

- 增加轻度平衡正则；
- 限制潜在通道容量；
- 在分析阶段引入 direct / latent 分解。

### 13.4 Rollout 数值爆炸

优先处理顺序：

1. 降低学习率；
2. 增强 range penalty；
3. 缩短早期 rollout horizon；
4. 在变换空间建模；
5. 必要时对预测增量做 clipping。

## 15. 推荐开发顺序

原规范最有价值的一点，是它明确规定了开发顺序。正确顺序应是：

1. 先做 persistence baseline；
2. 再做 observed-only MLP baseline；
3. 再做 observed-only 图模型；
4. 再上完整模型 + 1 个潜在节点；
5. 先确保一步训练稳定；
6. 再打开 rollout 训练；
7. 再补可视化；
8. 再做消融；
9. 最后才尝试多个潜在节点和更强生态先验。

这意味着我们的实施策略应该是“逐层验证”，而不是“一次性把所有组件堆满”。

---

## 16. 第一版项目的验收标准

只有同时满足下面几条，第一版才算完成：

1. 训练在时间切分数据上稳定；
2. 多步 rollout 优于 persistence 和 observed-only 基线；
3. 输出图结构和潜在通道相关可视化；
4. 去掉潜在通道后 rollout 明显变差；
5. 没有明显非生态数值伪影，比如大面积爆炸、NaN 或不合理负值。

如果只能做到“一步指标很好”，但 rollout 不行，那么这个项目还不能算达标。

## 17. 对后续实现工作的建议

如果下一步要真正开始编码，我建议按下面的工程路线推进：

### 第一步

先搭数据、预处理、窗口采样和 baseline。

### 第二步

补 observed-only 图模型，确认图结构本身是稳定可训练的。

### 第三步

加入单个潜在隐藏节点和 hidden -> observed 中介路径。

### 第四步

引入 rollout curriculum、主评估、可视化和消融。

### 第五步

只有当前四步全部跑通，再考虑更强先验、多隐藏节点和不确定性估计。

## 18. 一句话总结

这份规范本质上是在要求我们实现一个**面向多步动力学闭合的、具备潜在生态通道的图动力学模型**。第一版成功与否，不看模型是否复杂，而看它是否能在严格时间切分和 rollout 评估下，稳定且有说服力地优于 observed-only 基线。
